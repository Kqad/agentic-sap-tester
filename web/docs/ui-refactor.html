四、Power-user 交互（10 项新增） #

4.1 命令面板 Ctrl+K

模糊搜索三类目标：

  • 用例（标题 / TX 码 / 文件名）
  • 跳转（7 个 nav 目标）
  • 操作（清空缓存、切主题、切语言、退出登录、打开速查表）

打分：子序列匹配 + 前缀加权 + 相邻字符加权，模仿 VS Code 手感。 ↑/↓ 移动；↵ 触发；Esc 关闭。

4.2 键盘速查表 ?

快捷键动作

4.3 KPI 数字 count-up 动画

数字变化时以 ease-out-cubic 滚动，搭配 kpiBump Y 轴微弹（420ms）。DOM 复用：跨轮询不重建 cell，避免动画被打断。

4.4 通过率 KPI 内嵌 Sparkline

示例： （60×18 inline SVG，滑窗 K = 5–12，颜色随 KPI 语义，类似 SAPUI5 LineMicroChart）

4.5 步骤管线自动滚动

运行步骤推进时，当前 is-running 步骤平滑滚入视口。防抢手：用户在 1.5s 内手动滚动过 detail 区域时不抢。

4.6 日志条目入场微闪

新日志按 kind 着色闪入（550ms）：

  • info → 蓝半透明 → 透明
  • ok → 绿半透明
  • warn → 橙半透明
  • err → 红半透明 → 保留淡红背景

重渲（清空 / 过滤切换）时不触发入场动画。

4.7 状态徽章变化脉冲

用例 last-run 状态变化时，左侧 .tag 触发 wbTagPulse 缩放 + 阴影脉冲（550ms）。

4.8 Shift+Click 区段多选

rangeAnchorIdx 记住上次点击的可见行索引，Shift 点击时从 anchor 到当前行批量同步勾选状态。Anchor 行右侧蓝色小圆点提示。经典邮件客户端模式。

4.9 Run 完成 toast 通知

pollActiveRuns() 检测到刚结束的 run 时：

  1. 拉新 /api/results/recent 同步 KPI / 列表 / 结果表
  2. 弹出 toast：saptest1 ✓ passed · 41.6s
  3. 点 toast 直接跳到该用例的 History 标签

4.10 参数快捷编辑（详情面板内联）⭐

对应需求："比如 case 支持参数修改的那几个，一个小思路"

对包含 aiInput 步骤的用例，详情面板在步骤管线之上插入一个紧凑参数网格：

  • 每个 aiInput 步一格 → #order · 定位词 + 输入框
  • value 优先取 parsed.params[order]，否则取 exampleCode 中的 { value: "…" }
  • T Code 字段橙色 ⚠ 警示 — 改 T Code 会让 cache 失效
  • "保存参数"一键 PUT /api/cases/:id — 与完整编辑器走同一接口，apiGuide 字节级保留

七、可访问性 / 性能 #

  • 键盘可达：所有命令面板项、kbd cheat 都是 <button>/<a>
  • 不抢输入焦点：全局快捷键监听 tag in {input, textarea, select} 时直接 return
  • 滚动锚定：日志和步骤管线都有"用户滚动过则不强抢"判断
  • 轮询自适应：idle 时 4s 周期，有活跃 run 切 1.5s，结束自动回退
  • JS bundle 增量：约 +30KB 未压缩 / gzip 后约 +9KB
  • CSS 增量：约 +12KB 未压缩
  • 零外部依赖：vanilla DOM，没有引入 Motion / cmdk 等库

八、回滚 / 兼容性 #

  • 老 dashboard 保留为 VIEWS.dashboardLegacy，若需快速回退只需把 render() 里的渲染目标改回
  • 所有原路由完整保留，"完整编辑器"工作流（含 raw JSON 视图）一行未改
  • 后端任何接口都没动 → 部署只需替换 web/ 目录
  • 现有运行记录、cache 文件、用户、审计日志格式都没受影响
